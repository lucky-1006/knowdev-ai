## Quick Start Checklist

1. **Find an Issue**: Browse open issues or open a new one to discuss your ideas.
2. **Branch**: Branch off the `develop` (or `main`) branch following the format: `feat/your-feature` or `fix/your-fix`.
3. **Tests**: Check that backend and frontend tests pass before opening a PR.
4. **Pull Request**: Open a PR using our PR Template. Include description details and references to issues solved.
